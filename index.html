Domluvit pilot (20 min) Chci školení janAGI (open-source projekt) ↗

Jak spolupráce probíhá

  1. Scope call: vybereme proces, cíle a metriky pilotu.
  2. Návrh workflow: definujeme kroky, role a schvalování.
  3. Implementace: n8n + OpenClaw + konektory na vaše systémy.
  4. Pilotní provoz: sběr logů, ladění kvality a bezpečnosti.
  5. Předání: runbook, checklist, doporučení pro škálování.

Jak vypadá AI pilot v praxi (2 min)

Telegram / Email n8n workflow OpenClaw agent Schválení Zápis + audit log

Ukázkový průchod: Email assistant: návrh odpovědi → schválení odpovědnou osobou → odeslání + auditní záznam kroků a rozhodnutí.

Nabídka spolupráce

AI Pilot Sprint (pro firmu)

  • Výběr procesu + metriky
  • Implementace workflow (n8n) + agent layer (OpenClaw)
  • Schvalování + audit log
  • Předání + runbook

Rozsah: od 2–4 týdnů (upřesníme na callu).

AI Adoption Workshop (tým)

  • Role-based scénáře (obchod, backoffice, IT)
  • Praktické ukázky automatizací
  • Governance a pravidla používání AI

Rozsah: od 1 dne (online / onsite).

Delivery partner (retainer)

  • 1–2 procesy měsíčně
  • Průběžné ladění kvality, nákladů a bezpečnosti
  • Prioritizace backlogu a governance

Spolupráce: měsíční režim, kapacita dle plánu.

Co přesně dodám (deliverables pilotu)

  • n8n workflow (export) + popis uzlů
  • OpenClaw agent config (role, policy, nástroje)
  • Audit log / trace (kam se ukládá, co se loguje)
  • Runbook (provoz, restart, limity, fallback)
  • Checklist bezpečnosti (kde se data pohybují)
  • Doporučení pro škálování na další procesy

Pro koho to je

  • SME firmy s 1–3 klíčovými procesy na automatizaci
  • Týmy, které chtějí AI, ale potřebují kontrolu a audit
  • Organizace, kde je jasný owner procesu

Kdy se to nehodí

  • Když chcete „chatbota pro všechno“ bez procesního návrhu
  • Když není owner procesu nebo nikdo nechce schvalovat
  • Když není možné definovat cílové metriky pilotu

Use-cases podle oddělení

Obchod

Web research → CRM summary pro rychlejší follow-up.
Typický dopad: rychlejší podklady pro obchod a management.

Podpora

Triage ticketů → návrh odpovědi → schválení před odesláním.
Typický dopad: zrychlení odpovědí, méně chyb, dohledatelnost.

Backoffice

Faktury/objednávky → validace → zápis přes API s kontrolou.
Typický dopad: méně incidentů díky approvals a oprávněním.

Management

Týdenní reporty z dat → shrnutí + zdroje pro rozhodnutí.
Typický dopad: rychlejší rozhodování nad aktuálními daty.

HR

Screening CV → doporučení kandidátů + důvody.
Typický dopad: kratší shortlistování a jednotnější hodnocení.

IT

Change request → kontrola rizik → approval flow.
Typický dopad: stabilnější změnové řízení a auditní stopa.
